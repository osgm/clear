# Security Policy

感谢你帮助提升 `JoyCleaner` 的安全性。

## Supported Versions

当前默认仅对最新主干版本提供安全修复支持。

## Reporting a Vulnerability

请不要在公开 Issue 中直接披露漏洞细节。

建议通过私下渠道向维护者报告，并尽量提供：

- 漏洞类型与影响范围
- 复现步骤（PoC）
- 影响版本
- 可能的修复建议（如有）

在仓库公开发布前，请在此处补充安全联系邮箱。

## Response Process

维护者收到报告后，将按以下流程处理：

1. 确认收到并评估影响
2. 复现并确定修复方案
3. 发布修复版本与安全公告（必要时）
4. 在合理时间内公开披露细节

## Scope Notes

以下内容通常不视为安全漏洞：

- 不影响安全边界的 UI 问题
- 仅限本地环境且无越权风险的行为
- 已明确记录的使用限制
